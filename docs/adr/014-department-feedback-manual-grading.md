# ADR-014: Обратная связь кафедры — документы без машинной валидации, оценка только преподавателем

## Статус
Изменён ADR-016. Историческое решение о полном отказе от машинной проверки больше
не действует: подробная сверка сохранена как подсказка преподавателю, но автоматической
итоговой оценки по-прежнему нет. Режим `FREE_TEXT` отдельно отменён ADR-015.

## Контекст
На момент решения была зафиксирована обратная связь кафедры по модели работы с
документами и отчётом. Детальные поля документов, терминология и эталонный кейс
остаются отдельными внешними входами. Ключевое:
- «Машинную оценку мы не внедряем, оценка и проверка полностью на преподавателе».
- Документ: курсант сначала ВЫБИРАЕТ правильный среди обманок, затем ЗАПОЛНЯЕТ;
  заполнение — либо «свободный ввод» (норма, ради практического навыка), либо набор
  именованных полей; режим задаёт преподаватель в Конструкторе.
- Нужна ПЕЧАТЬ заполненного и ПРИКРЕПЛЕНИЕ справочных документов (формы/акты) для
  просмотра курсантом во время работы.
- Этап «Среда»: добавить выбор методов обследования (смыв, отбор воды, измерение
  температуры и т.д.).
- Эталонный кейс кафедра делает сама; наш шигеллёзный кейс — внутренний фикстур для E2E.

## Решение
1. Никакой машинной оценки. Сверка (grade_case с findings «верно/неверно»,
   InspectionCheck, сверка полей и выбора документа) НЕ используется как оценщик. Все
   действия курсанта (выбор документа, заполнение, путь развилки, вывод осмотра) только
   ЗАПИСЫВАЮТСЯ и попадают в отчёт. Вердикт ставит преподаватель.
2. Документ-задание = выбор из списка с обманками (остаётся) + заполнение в одном из двух
   режимов: FREE_TEXT (многострочный свободный ввод) | FIELDS (именованные поля). Режим
   задаёт преподаватель. Ожидаемые значения полей и «правильность» машиной не проверяются
   (пометка «правильный документ» сохраняется для преподавателя в отчёте, не как авто-вердикт).
3. Печать заполненного документа и прикрепление справочных ассетов к заданию (просмотр курсантом).
4. Этап «Среда»: к хотспотам/действиям добавляется выбор метода обследования
   (переиспользует «иконки действий» из ADR-013).
5. Отчёт = нейтральный печатный транскрипт: ответы по этапам + выбранный путь развилки +
   идентичность (блок N3). Без галочек и без общего балла.

## Последствия
- Режется самая тяжёлая логика: валидация документов и авто-оценка. Чистый минус по объёму.
- epicase-document-templates переписывается: «выбор из обманок + два режима ввода, без сверки
  значений». epicase-stage-mechanics синхронизируется (поиск/осмотр/развилка записываются,
  не оцениваются).
- Доменная модель документа: добавляется fill_mode + (для FIELDS) перечень полей без ожидаемых
  значений; декой-набор остаётся; добавляются ссылки на справочные ассеты. Слом формата
  допустим (архивов в обращении нет, FORMAT_VERSION=1, эталон ещё не получен).
- grade_case/findings не удаляются немедленно, но перестают рендериться как вердикт; отчёт
  показывает сырые ответы. Блок N3 (идентичность) сохраняется.
- Печать — QTextDocument/QPrinter (локально, без сети).
- К5: «Оценка СЭС» и «рапорт командира части» подтверждены; «донесение» не используется.
- К4: эталонный кейс за кафедрой; наш шигеллёзный кейс + изображения — фикстур для E2E.
