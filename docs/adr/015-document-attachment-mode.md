# ADR-015: Документ-задание — режим «Прикрепление файла» вместо свободного ввода

## Статус
Принят. **Амендмент к ADR-014** (пункт 2): режим `FREE_TEXT` заменяется режимом
`ATTACHMENT`. Политику оценивания уточняет ADR-016: машинная сверка остаётся подробной
подсказкой преподавателю, но не формирует итоговый балл или pass/fail.

## Контекст
Свободный ввод (многострочное текстовое поле) на практике неудобен и нереалистичен для
заполнения официального бланка (Форма 23, акт эпидобследования). Решение продукт-оунера:
курсант заполняет НАСТОЯЩИЙ бланк в своём редакторе (Word/PDF) и ПРИКРЕПЛЯЕТ готовый файл;
преподаватель открывает вложение при проверке.

## Решение
1. `FillMode` = {`ATTACHMENT`, `FIELDS`}. `ATTACHMENT` («Прикрепить файл») заменяет
   `FREE_TEXT` и становится нормой для практического навыка. `FIELDS` (именованные поля)
   остаётся без изменений.
2. Курсант прикрепляет СВОИ файлы с нуля. Бланк от кейса не выдаётся; механика
   `reference_assets` (шаблон-заготовка от кейса) для этого режима не используется — это
   отдельная, отложенная идея.
3. Каждый блок `ATTACHMENT` принимает от 0 до 10 файлов. Настройки «один файл» /
   «несколько файлов» нет.
4. Прикреплённые файлы сохраняются в архив результата `.epiresult` (переиспользуется
   существующий asset-канал результата). Преподаватель открывает/сохраняет их из отчёта.
5. Вложения не проверяются автоматически по содержимому: отчёт показывает факт
   прикрепления, имя и размер файла и позволяет преподавателю открыть или сохранить
   содержимое. Содержательную оценку выполняет преподаватель (ADR-016).

## Последствия
- Домен документа: значение `FREE_TEXT` убирается из `FillMode`, добавляется `ATTACHMENT`;
  в `DocumentResponse` поле `free_text` заменяется списком вложений (asset-id). Поле
  `allow_multiple` сохраняется в сериализации формата v1 только для обратной совместимости;
  Player его игнорирует.
- Архив результата: модель попытки несёт asset-id вложений; Player кладёт байты вложений в
  asset-map перед `write_epiresult`, а Constructor получает их при чтении `.epiresult`.
- Player: текстовое поле свободного ввода заменено виджетом прикрепления
  (`QFileDialog`) с постоянным множественным выбором и лимитом 10 файлов на блок; имена
  вложений видны курсанту.
- Constructor: пункт тумблера «Свободный ввод» заменён на «Прикрепить файл»;
  настройки «один файл» / «несколько файлов» нет.
- Печать заполненного в приложении (ADR-014, п.3) для `ATTACHMENT` теряет смысл (курсант
  приносит готовый файл); остаётся опциональной для `FIELDS`, приоритет понижается.
- Прикрепление справочных ассетов кейсом (ADR-014, п.3) — отложено.
