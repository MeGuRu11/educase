## Принцип

Скиллы кодируют **контекст и правила**, чтобы агенты не переоткрывали закрытые вопросы.
Параллельно работает **лестница агентов** (`.claude/agents/`): cheapest capable model на задачу.

| Агент              | model  | Роль                                   |
|--------------------|--------|----------------------------------------|
| architect          | opus   | архитектура, ADR, ревью дизайна        |
| senior-developer   | opus   | критичный код (домен, кодеки, движок)  |
| ui-builder         | sonnet | PySide6, некритичный UI, рендереры     |
| code-reviewer      | sonnet | ревью PR по quality gate               |
| reality-checker    | sonnet | ручная регрессия, визуальные пруфы     |

Codex GPT 5.5 — отдельный инструмент (скаффолдинг), правила в `CODEX.md`.

## Скиллы (реализованы как заготовки)

1. **educase-project** — мастер-контекст: что за проект, архитектура, quality gate, жёсткие
   правила. Триггер: любая работа с кодом проекта.
2. **educase-stage-mechanics** — шесть этапов, ветвление «Вариант B», строгий поиск, механика
   документов, осмотр помещений. Триггер: движок кейса, рендереры этапов.
3. **educase-document-templates** — шаблоны ведомственных документов (ДМ4, приложения 1/3/22/23,
   акт расследования) и механика проверки полей. Триггер: реализация документов.
4. **educase-archive-format** — формат `.epicase`/`.epiresult` (ZIP + manifest.json + ассеты),
   запрет показа JSON, запрет сети. Триггер: импорт/экспорт, кодеки, ассеты.
