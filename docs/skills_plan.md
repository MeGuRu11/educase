# EpiCase — проектные skills

Проект поддерживает одинаковый набор из четырёх предметных skills:

- `.agents/skills/epicase-*` — инструкции Codex;
- `.claude/skills/epicase-*` — инструкции Claude Code.

Skills кодируют действующие ограничения и доменные правила. Архитектурным источником
истины остаются последние принятые ADR и фактический код; при изменении решения обе
копии соответствующего skill обновляются вместе.

## Текущий набор

1. `epicase-project` — архитектурные границы, файловый обмен, отсутствие сети,
   quality gate и политика предварительной машинной проверки ADR-016.
2. `epicase-stage-mechanics` — шесть этапов, неблокирующее ветвление, строгий поиск,
   осмотр, документы и таймлайны.
3. `epicase-document-templates` — режимы `FIELDS`/`ATTACHMENT`, правила полей,
   предварительная сверка и ведомственные документы.
4. `epicase-archive-format` — `.epicase`/`.epiresult`, ZIP-контейнер, manifest,
   checksum, ассеты и ошибки валидации.

## Роли агентов

Конфигурации ролей находятся в `.codex/agents/` и `.claude/agents/`:

| Роль | Уровень | Зона ответственности |
|---|---|---|
| `architect` | opus | архитектура, ADR, границы слоёв |
| `senior-developer` | opus | домен, кодеки, движок прохождения |
| `ui-builder` | sonnet | PySide6 UI и рендереры |
| `code-reviewer` | sonnet | ревью по quality gate |
| `reality-checker` | sonnet | ручная регрессия и визуальные доказательства |

Правила отдельного инструмента Codex GPT 5.5 для скаффолдинга находятся в `CODEX.md`.

## Расширение набора

Новый skill добавляется только для устойчивой предметной области, которая не помещается
в существующие четыре без смешения ответственности. Упаковка PyInstaller и общие приёмы
pytest-qt пока покрываются `packaging/`, README и ролью `reality-checker`; отдельные skills
для них не требуются.
