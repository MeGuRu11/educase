# N5: инфраструктурные иконки хотспотов — дизайн

**Дата:** 2026-06-30
**Статус:** визуальное направление A утверждено пользователем
**Область:** application draft, общий UI, Constructor и Player

## Цель

Заменить постоянно видимые прямоугольники хотспотов в Player на картографические
teal-пины с белой инфраструктурной пиктограммой и короткой постоянной подписью.
Прямоугольная геометрия ADR-013 сохраняется как область клика и редактирования.

Constructor должен позволять преподавателю явно выбрать пиктограмму для каждой
зоны и сразу показать её на холсте.

## Утверждённое направление

Выбран вариант A «Картографический пин»:

- круглый teal-маркер с небольшим указателем вниз;
- белая контурная пиктограмма инфраструктуры;
- тёмная контрастная подпись под маркером;
- цвета только из `theme.qss`: `#0F766E`, `#0B5E57`, `#17393A`,
  `#D9EEEB`, `#FFFFFF`;
- исходная прямоугольная зона в Player по умолчанию не видна;
- при наведении реальная область клика временно получает лёгкую teal-подсветку.

Маркер размещается в центре `HotspotShape`. Подпись показывается снизу, а у
нижней границы изображения — сверху. Она ограничивается двумя строками и
шириной 160 пикселей; полный текст остаётся в tooltip.

## Каталог пиктограмм

В MVP используется фиксированный локальный каталог:

| Ключ | Назначение |
|---|---|
| `inspect` | универсальный объект осмотра и fallback |
| `barracks` | казарма, жилое помещение |
| `canteen` | столовая, пищеблок |
| `medical` | медпункт, изолятор |
| `water` | водоснабжение, водонапорная башня |
| `sanitary` | умывальная, туалет, санитарный узел |
| `storage` | складское помещение |
| `cold_storage` | холодильная камера |
| `waste` | отходы, мусоросборник |
| `entrance` | вход, КПП |

Преподаватель выбирает тип вручную. Автоподбора по подписи, fuzzy-матчинга,
пользовательской загрузки SVG и сетевых каталогов нет.

Неизвестный или пустой ключ безопасно отображается как `inspect`. При открытии
кейса на редактирование неизвестное исходное значение сохраняется в скрытых
данных элемента выбора и не теряется при сохранении, пока преподаватель явно не
выберет другую пиктограмму. Legacy-ключ `zoom` отображается через fallback.

## Архитектура

Домен и формат архива уже содержат `Hotspot.icon: str`; менять
`SchemeDocument`, ADR-013 и `format_version` не требуется.

Поток данных:

```text
Constructor ZonePropsCard
        ↓ icon key
HotspotDraft.icon
        ↓ build/load
Hotspot.icon
        ↓ archive round-trip
Player hotspot renderer
```

`HotspotDraft` получает поле `icon`. `_build_hotspots` и
`_hotspot_to_draft` копируют значение в обе стороны, включая вложенные виды.

Общий реестр ключей, подписей и package resources размещается в
`epicase_ui`. Constructor и Player используют его как presentation-компонент.
Core не зависит от `epicase_ui`; UI не импортирует Infrastructure.

SVG хранятся локально в пакете. Значение `Hotspot.icon` используется только как
ключ allowlist-реестра и никогда не интерпретируется как путь к файлу.

## Constructor

В `ZonePropsCard` добавляется поле «Иконка объекта» на базе `QComboBox`.
Элементы содержат SVG-превью, русское название и стабильный строковый ключ.
Новая зона по умолчанию получает `inspect`.

На `SchemeZoneCanvas` прямоугольник, номер и восемь ручек остаются: это
редактор области клика, а не финальное отображение Player. В центре зоны
показывается выбранный пин. Изменение combo обновляет соответствующий
`ZoneItem` без пересоздания зоны и без изменения её координат.

Поведение одинаково для корневых и вложенных видов. Смена фона, загрузка кейса,
добавление, удаление и перенумерация зон сохраняют существующую семантику.

## Player

Player больше не рисует постоянную рамку и заливку `QGraphicsRectItem`.
Для каждого хотспота создаётся визуальный маркер:

- позиция — центр нормализованного `HotspotShape`;
- размер пина и подписи не меняется при зуме;
- белая пиктограмма берётся из общего реестра;
- подпись остаётся читаемой на светлом и тёмном фоне;
- неизвестный ключ использует `inspect`;
- tooltip содержит полную подпись.

Hit-test остаётся по `HotspotShape.contains`, поэтому кликабельная область не
уменьшается до размера иконки. Клик внутри зоны по-прежнему открывает `child`
или reveal-диалог. Зум, панорама, стек вложенных видов и оценивание осмотра не
меняются.

При наведении на область хотспота показывается тонкая teal-рамка с лёгкой
заливкой; после ухода она исчезает. Если зоны перекрываются, сохраняется текущий
приоритет первого хотспота в кортеже.

## Ошибки и совместимость

- старый кейс без `icon` открывается с универсальной пиктограммой;
- неизвестный ключ не вызывает исключение и не загружает произвольный файл;
- отсутствующий или битый SVG отдельной пиктограммы включает векторный
  fallback `inspect`;
- отсутствие фонового изображения по-прежнему показывает существующий
  плейсхолдер;
- формат `.epicase` остаётся версии 1;
- `InspectionCheck`, ответы курсанта и отчёт не меняются.

## Вне области N5

- изменение прямоугольной модели ADR-013;
- полигоны, стены, слои и редактор графа;
- автоматическое распознавание объектов на изображении;
- пользовательские наборы иконок;
- collision avoidance для пересекающихся подписей;
- изменение reveal-диалога, документов или правил оценивания.

6. Реализовать Constructor; полный gate и commit.

   fallback, hover, hit-test и зум.
8. Реализовать Player; полный gate и commit.
9. Обновить `TASKS.md`, выполнить финальный gate и Conventional Commit.

Каждый production-шаг начинается с запуска нового целевого теста и фиксации
ожидаемого падения. Сразу после минимальной реализации выполняется полный
зелёный gate до следующего изменения:

1. `ruff check src tests`;
2. `mypy src tests`;
3. `pytest -q`;
4. `python -m compileall -q src tests`.

## Живая приёмка

В Constructor проверить:

1. у новой зоны выбран универсальный тип;
2. смена типа сразу меняет пин в центре зоны;
3. перемещение и resize не меняют выбранную пиктограмму;
4. корневые и вложенные зоны сохраняются и повторно открываются с теми же
   иконками;
5. рамки, ручки и номера остаются удобными для редактирования.

В Player проверить:

1. на схеме постоянно видны пины и подписи, но не прямоугольные рамки;
2. пины читаются на светлых и тёмных участках;
3. наведение кратко показывает настоящую область клика;
4. клик по любой точке старой прямоугольной зоны работает;
5. zoom/pan не меняет экранный размер пина и не ломает hit-test;
6. вложенный вид и кнопка «Назад» работают как раньше;
7. длинная подпись переносится максимум на две строки и имеет полный tooltip;
8. старый кейс без иконок показывает универсальные маркеры.

## Критерий завершения

N5 завершена, когда Constructor сохраняет выбранные типы инфраструктуры,
Player отображает вариант A с прежней областью клика, старые кейсы безопасно
работают, полный gate зелёный, `TASKS.md` обновлён и пользователь выполнил
живую проверку.
