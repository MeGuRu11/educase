# ICON-1B: исходники и Windows ICO

## Цель

Подготовить две различимые фирменные иконки EpiCase для Windows:

- Constructor читается как инструмент проектирования;
- Player читается как маршрут прохождения;
- обе иконки сохраняют общую шестигранную геометрию A2 и палитру проекта;
- различие остаётся заметным на размере 16×16.

Подключение иконок к `QApplication`, окнам, панели задач и PyInstaller относится
к следующей задаче `ICON-1C`.

## Визуальная система

Обе иконки используют прозрачный квадратный холст и шестигранник A2.

### Constructor

- белый шестигранник;
- контур `#17393A`;
- в полной версии — светлая чертёжная сетка `#D9EEEB`;
- треугольная схема с тремя латунными точками `#B49A56`;
- на 16–24 px сетка удаляется, линии и точки укрупняются.

### Player

- залитый teal-шестигранник `#0F766E`;
- контур `#17393A`;
- белый изогнутый маршрут;
- светлая стартовая точка `#D9EEEB`;
- направляющая стрелка;
- на 16–24 px маршрут и стрелка упрощаются и утолщаются.

Буквы, градиенты, текстуры и новые цвета не используются.

## Ресурсы

Новые файлы размещаются в `src/epicase_ui/resources/app_icons/`:

- `epicase_constructor.svg`;
- `epicase_constructor_small.svg`;
- `epicase_constructor.ico`;
- `epicase_player.svg`;
- `epicase_player_small.svg`;
- `epicase_player.ico`.

Существующие SVG стартового экрана в `resources/brand/` не изменяются.

Полные SVG используются для кадров от 32 px. Упрощённые SVG используются для
16, 20 и 24 px.

## Состав ICO

Каждый ICO содержит PNG-кадры следующих размеров:

`16, 20, 24, 32, 40, 48, 64, 128, 256`.

Кадры имеют прозрачный фон и глубину 32 bpp. Размер 256 записывается в таблице
ICO нулевым байтом ширины и высоты согласно формату.

## Воспроизводимая генерация

Скрипт `tools/generate_app_icons.py`:

1. читает только allowlist из четырёх SVG;
2. валидирует исходник через `QSvgRenderer`;
3. рендерит каждый размер в прозрачный `QImage`;
4. кодирует кадр в PNG;
5. собирает ICO средствами стандартной библиотеки;
6. атомарно заменяет итоговые файлы после успешной сборки обоих ICO.

Используются только PySide6 и стандартная библиотека. Новые runtime- или
dev-зависимости не добавляются.

`pyproject.toml` включает `resources/**/*.ico` в package data, чтобы ресурсы
были готовы к `ICON-1C` и будущей упаковке.

## Ошибки и безопасность

- неизвестные имена приложений и произвольные пути генератор не принимает;
- отсутствующий или невалидный SVG завершает генерацию с понятной ошибкой;
- частично собранный ICO не заменяет существующий ресурс;
- генератор не обращается к сети.

## Тестирование

Тесты проверяют:

- валидность четырёх SVG через `QSvgRenderer`;
- точный набор и порядок размеров в обоих ICO;
- корректные заголовки ICO и границы каждого кадра;
- PNG-сигнатуру, размеры и alpha-канал каждого кадра;
- совпадение checked-in ICO с повторной генерацией по структуре и DPI;
- максимальное отклонение RGBA-канала не более 2 из 255 из-за платформенного
  округления антиалиасинга;
- наличие SVG и ICO в package data.

PNG-сжатие не сравнивается побайтно: оно может различаться между версиями Qt,
не меняя изображение. Растеризация Qt также может отличаться на 1–2 уровня
RGBA-канала между графическими платформами.

После реализации выполняется полный gate:

1. `ruff check src tests`;
2. `mypy src tests`;
3. `pytest -q`;
4. `python -m compileall -q src tests`.

## Критерии завершения

`ICON-1B` завершена, когда:

- обе пары full/small SVG утверждённого варианта A добавлены;
- оба ICO содержат все девять размеров;
- checked-in ICO воспроизводятся генератором с визуально неразличимым
  отклонением каналов не более 2 из 255;
- тесты и полный quality gate зелёные;
- в `TASKS.md` закрыта только `ICON-1B`.

## Живая проверка

Открыть оба ICO в Проводнике Windows и проверить:

1. различимость Constructor и Player в режимах мелких и крупных значков;
2. читаемость на светлом и тёмном фоне;
3. отсутствие белого квадратного фона вокруг шестигранника;
4. сохранение формы без размытия на 16, 24, 32, 48 и 256 px.
