# PKG-2: Windows-установщик EpiCase

## Цель

Собирать один автономный установщик EpiCase для компьютерного класса. Пользователь
выбирает один из трёх вариантов:

- оба приложения — вариант по умолчанию;
- только Constructor;
- только Player.

## Технология и внешний вид

Используется стабильный Inno Setup 6.7.3. Скрипт установщика хранится в
`packaging/installer.iss`.

Установщик использует `WizardStyle=modern dynamic`: стандартный современный
интерфейс Inno Setup автоматически соответствует светлой или тёмной теме Windows.
Интерфейс и названия вариантов установки — на русском языке.

## Установка

Установка выполняется для всех пользователей с запросом прав администратора:

- каталог — `{autopf}\EpiCase`;
- архитектура — x64;
- в состав входят готовые one-file EXE из `dist/`;
- выбранные приложения получают общие ярлыки в папке `EpiCase` меню «Пуск»;
- ярлыки на общем рабочем столе создаются только по выбору пользователя;
- для каждого ярлыка используется иконка соответствующего приложения.

Установщик имеет стабильный `AppId`, имя `EpiCase`, версию из `pyproject.toml` и
штатный деинсталлятор Windows.

## Выбор и повторная установка

Три фиксированных setup type исключают пустой выбор компонентов. Каждому типу
соответствует точный набор компонентов Constructor/Player.

При повторном запуске установщика выбранный набор становится фактическим:
EXE и ярлыки снятого компонента удаляются, выбранные компоненты устанавливаются
или обновляются. Пользовательские `.epicase` и `.epiresult` файлы не затрагиваются.

## Сборка

Корневой `build_installer.bat`:

1. вызывает `build_all.bat`;
2. ищет `ISCC.exe` в `PATH` и стандартных каталогах Inno Setup 6;
3. завершает работу с понятной ошибкой, если компилятор отсутствует;
4. компилирует `packaging/installer.iss`;
5. создаёт `dist/EpiCase Installer 0.1.0.exe`.

Скрипт не загружает и не устанавливает зависимости. Inno Setup устанавливается
разработчиком один раз отдельно.

## Проверка

Автоматические тесты проверяют:

- совпадение версии установщика с `pyproject.toml`;
- три setup type и отсутствие пустого выбора;
- привязку EXE и ярлыков к правильным компонентам;
- x64, административную установку и современный динамический стиль;
- стабильный `AppId`, деинсталлятор и очистку снятого компонента;
- fail-fast поведение `build_installer.bat`.

Живая проверка включает компиляцию установщика, тихую установку каждого из трёх
вариантов в изолированный тестовый каталог, проверку установленных файлов и тихое
удаление.

## Вне scope

Цифровая подпись, MSI, GPO-развёртывание, автообновление, загрузка компонентов из
сети и изменение ассоциаций `.epicase`/`.epiresult` не добавляются.
