# UI readability and attachment cards

## Статус

Подтверждено пользователем 28 июня 2026 года.

## Цель

Улучшить читаемость названий интерактивных зон на фотографиях и привести отображение
прикреплённых файлов в Player и отчёте Constructor к компактному карточному виду.

Доменная модель, архивы `.epicase`/`.epiresult`, правила проверки и содержимое вложений
не меняются.

## Этап 1 — подписи хотспотов

- Бирюзовая рамка и полупрозрачная заливка зоны сохраняются.
- Название помещается на тёмную контрастную плашку внутри верхней части зоны.
- Текст белый, полужирный, с внутренними отступами.
- Длинное название переносится по ширине зоны; полное название остаётся в tooltip.
- Геометрия зоны, hit-test, зум, панорамирование и переход во вложенный вид не меняются.
- Графические цвета задаются через API `QGraphicsItem`, поскольку QSS к нему
  неприменим; значения соответствуют палитре `theme.qss`.

## Этап 2 — карточки вложений

### Player

- Большой пустой `QListWidget` заменяется компактным контейнером.
- Пока файлов нет, контейнер показывает короткое нейтральное сообщение.
- Каждый файл отображается отдельной карточкой: расширение, имя, размер и кнопка
  удаления.
- Для нескольких файлов доступно удаление по одному; «Очистить всё» видно только при
  непустом списке.
- Выбор одного файла по-прежнему заменяет предыдущий, множественный выбор добавляет
  файлы.

### Отчёт Constructor

- Заголовок секции показывает число вложений.
- Каждое вложение отображается карточкой: расширение, имя, размер, «Открыть» и
  «Сохранить как…».
- Если байтов файла нет в архиве, карточка показывает предупреждение, а действия
  отключены.

### Стиль

Player и Constructor используют одинаковые `objectName` и правила в `theme.qss`:
белая поверхность, рамка `#D4DAE0`, фон секции `#F6F8FA`, бирюзовый badge
`#D9EEEB/#0F766E`, основной текст `#1F2A33`, вторичный `#66727E`.

UI-компоненты остаются раздельными: приложения не импортируют виджеты друг друга.

## Автоматические проверки

- Сцена схемы содержит контрастную плашку, белый текст и ограничение ширины подписи.
- Длинная подпись сохраняется в tooltip и не меняет hit-test зоны.
- Player создаёт карточки по числу файлов, показывает размер и поддерживает удаление
  одного файла и очистку всех.
- Отчёт показывает badge, размер и количество вложений; отсутствие байтов отключает
  действия.
- Существующие тесты записи, открытия и сохранения вложений остаются зелёными.

## Живая проверка

На `shigellosis.epicase` проверить:

1. Названия всех зон читаются на светлых и тёмных участках схемы, включая
   «Водонапорная башня».
2. Зум, перетаскивание, клики и вложенные виды работают как раньше.
3. В Player один и несколько файлов занимают только необходимую высоту; имя, тип,
   размер и удаление читаемы.
4. В отчёте карточки файлов визуально отделены, а «Открыть» и «Сохранить как…»
   продолжают работать.
5. Отсутствуют новые баллы, вердикты или анализ содержимого вложений.

## Поставка

1. Отдельный коммит для подписей хотспотов.
2. Отдельный коммит для карточек вложений и QSS.
3. Полный quality gate и ручная проверка на реальном кейсе.
