# Механика этапов EpiCase

Источник структуры — `СХЕМА.xlsx` (спецификация кафедры). Шесть фиксированных этапов, без
редактора графа узлов (ADR-004). Повторяющийся приём интерфейса — диалог **«что хотите
сделать?»** (этапы 2, 4, 5, 6).

## Шесть фиксированных этапов
1. **Пациенты** — программа проводит **контекстный поиск по ключевым словам**.
   - Ячейки под каждого пациента: **клинические данные, эпид. анамнез, ключевые слова**.
   - Пациенты **не связаны между собой** в программе.
   - Предусмотреть запуск **без пациентов** (сразу с эпид. диагноза) + ключевые слова,
     задающие эпид. направленность отдельных разделов.
2. **Клинико-эпидемиологический диагноз** — точка ветвления: **правильный / неправильный путь**
   (по правильным ключевым словам); диалог «что хотите сделать?».
   - Документы: **Внеочередное донесение ДМ4**, **форма 23 (Ф23)**.
3. **Обследование контактных лиц** — **ячейки под личный состав на общей схеме каждого объекта
   + иконки** действий.
4. **Обследование объектов внешней среды** — диалог «что хотите сделать?».
   - Документы: **Акт ГСЭН (для МПД)**, **рапорт командира части** + другие неправильные (обманки).
   - **Общая схема объектов внешней среды с фото** + блок иконок действий.
5. **Оценка СЭС** (санитарно-эпидемиологического состояния) — контекстный поиск; диалог.
   - Документы: **План + приказ** + другие неправильные (обманки).
   - Критерии и пороги уровней — в `epicase-document-templates` (Приложение 22).
6. **Постановка заключительного эпид. диагноза** — контекстный поиск; диалог.
   - Документ: **Акт расследования очага** (18 разделов) + обманки.
   - **Сроки наблюдения за очагом** (таймлайны наблюдения).

## Ветвление «Вариант B» (этап 2)
Оба пути (верный/неверный) проходят одни и те же этапы. Ошибка **не блокирует** прохождение и
вскрывается только в финальном отчёте (ADR-005).

## Поиск по ключевым словам
Строгое сопоставление с синонимами, которые задаёт преподаватель. **Никакого fuzzy-матчинга,
никакой устойчивости к опечаткам** (ADR-006).

## Механика документов («Вариант B», ADR-007)
Курсант (а) выбирает правильный документ из списка с обманками И (б) заполняет его поля;
поля сверяются с ожидаемыми значениями. Шаблоны и поля — в `epicase-document-templates`.

## Осмотр помещений (виртуальный обход, ADR-008 + ADR-012)
Двухуровневая навигация: план объекта с кликабельными входами → интерьерные фото с хотспотами
→ свободный вывод курсанта, проверяемый по ключевым словам преподавателя. Этапы 3 и 4, MVP.

Схема объекта — это `SchemeDocument` (см. DATA_MODEL.md): слои с комнатами (прямоугольники/
полигоны), стенами, маркерами (входы / личный состав / иконки действий). **Один тип на два
представления:** Constructor — встроенный векторный **редактор** (фазы R1→R2→R3), Player —
**вьюер только-чтение**. Внешние инструменты не используются.
